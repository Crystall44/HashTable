# Хеш-таблица, множество, мультимножество на C

Реализация хеш-таблицы с разрешением коллизий методом цепочек, а также
структур «множество» (Set) и «мультимножество» (Multiset) на её основе.

Учебный проект в рамках практики по структурам данных.

# Что реализовано

Хеш-таблица (HashTable)

Структура данных, хранящая пары ключ-значение. Обеспечивает
быстрый доступ к значению по ключу — в среднем O(1).

Операции:  
ht_put — вставка пары (ключ, значение)  
ht_get — получение значения по ключу  
ht_contains — проверка наличия ключа  
ht_remove — удаление по ключу  
ht_size — количество элементов

Множество (Set)

Набор уникальных элементов без повторений. Основано на хеш-таблице:
ключом является сам элемент, значением - маркер присутствия.

Операции:  
set_add — добавить элемент  
set_remove — удалить элемент  
set_contains — проверить наличие  
set_union — объединение двух множеств  
set_intersection — пересечение двух множеств  
set_difference — разность двух множеств

Мультимножество (Multiset)

Множество, допускающее повторяющиеся элементы. Хранит счётчик
вхождений для каждого элемента.

Операции:  
multiset_add — добавить элемент N раз  
multiset_remove — удалить элемент N раз  
multiset_count — получить количество вхождений  
multiset_contains — проверить наличие хотя бы одного

# Хеш-функция

Используется FNV-1a (Fowler–Noll–Vo), 32-битный вариант.

Принцип работы:

Для каждого байта ключа выполняются две операции:  
1. XOR текущего хеша с байтом: hash ^= byte  
2. Умножение на простое число: hash *= 16777619

Начальное значение: 2166136261.

Почему FNV-1a:

FNV-1a была разработана специально для хеш-таблиц и протестирована на реальных данных. Её главные достоинства:

Простота. Вся функция - это цикл из двух операций (XOR и умножение).

Скорость. На каждый байт ключа выполняется всего одно умножение - одна из самых быстрых операций процессора

Хорошая лавинность. Умножение на большое простое число (16777619) «размазывает» биты так, что даже маленькое изменение ключа даёт совершенно другой хеш.

Работа с любыми данными. Функция оперирует сырыми байтами, поэтому неважно, что выступает ключом — строка, число или бинарные данные.

Детерминированность. Одинаковый ключ всегда даёт одинаковый хеш на любой платформе.

# Разрешение коллизий

Используется метод цепочек:
  
Каждая ячейка таблицы хранит связный список элементов  
При коллизии новый элемент добавляется в начало списка  
Поиск идёт только по цепочке в одной ячейке, а не по всей таблице
